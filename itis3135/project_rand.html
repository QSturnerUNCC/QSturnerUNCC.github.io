Requirements & Design for Proposed Project

Project Overview

  • Application Description and Overview:
    • The P.H.A.T. Tags website is the primary location and digital headquarters of the “P.H.A.T. Tags” business. The website will provide information on products the business offers as well as a way to purchase said products. Information about the business’s founding, the people involved in the business, and contact information.
    • The intended users of the website would be potential buyers of P.H.A.T. products as well as returning customers.
  • Intended Users:
    • Anyone interested in the products
    • Aged approx. 18-99
  • Client Information
    • Names: Andersen Sturner, Shawn Reece
    • Organization/Business: P.H.A.T.
    • Email address: shawncrisr@hotmail.com

  • Page Design
    • Main Page (index):
      • Purpose: To give the users a good starting off place on the website.
      • Audience/users: All
      • Content:
        • Navigation bar
          • Hamburger Icon (navigation)
            • Links to all main pages
          • Search bar/icon
          • Cart icon
        • Image Carousel
        • Featured Products
      • Data Entry: None
      • Buttons, Hyperlinks, Dropdowns:
        • Search button
        • Cart button
        • Links to main pages
      • Actions:
        • Clicking navigation links
        • Search based on user value
        • Scrolling down
      • Special Notes: None
    • Sign In:
      • Purpose: To allow the user to create an account or sign in to access exclusive benefits and check out faster
      • Audience: All
      • Content:
        • Create Account Page:
          • Textboxes for "email", "set password", and "confirm password"
          • Checkbox to receive email offers
          • "Sign Up" button
        • Sign In Page:
          • Textboxes for email and password
          • Checkbox to remember login information
          • "Sign In" button
      • Data Entry: Yes
      • Data Validation: Yes
      • Buttons, Hyperlinks, Dropdowns:
        • "Sign Up" button
        • "Sign In" button
      • Actions:
        • Button click
        • Data (login information) processing
      • Special Notes: None
    • Contact Us:
      • Purpose: To present contact information for the company to the user
      • Audience: Users who are interested in contacting the business in which the website belongs to
      • Content:
        • Contact information
        • "Email Us" button
        • Hyperlinks to emails and phone numbers
      • Data Entry: None
      • Buttons, Hyperlinks, Dropdowns:
        • Email hyperlinks
        • Telephone number hyperlinks
        • "Email Us" button
      • Actions: Clicking buttons
      • Special Notes: None
    • Shop Now:
      • Purpose: To show products available for purchase and information on said products
      • Audience: Users who want to shop for P.H.A.T. products
      • Content:
        • Clickable images of products
        • Sort button
        • Filter button
        • Navigation bar with various product categories
        • "Check Out" button
        • Shop search bar
        • Page number buttons
        • Next/Previous button
      • Data Entry: Yes
      • Data Validation: None
      • Buttons, Hyperlinks, Dropdowns:
        • Image links
        • Sort button/dropdown
        • Filter button/dropdown
        • Shop navigation bar
        • "Check Out" button
        • Page number buttons
        • Next/Previous button
      • Actions:
        • Clicking page number/next/previous page buttons
        • Clicking Sort button/dropdown
        • Clicking Filter button/dropdown
        • Clicking "Check Out" button
        • Search based on product name or description
      • Special Notes: None
    • Cart:
      • Purpose: To allow users to view the items placed in their cart and check out
      • Audience: Users prior to purchasing products
      • Content:
        • Table of products:
          • Picture of product
          • Product name
          • Price
          • Quantity
          • Subtotal
          • Total
        • "Update Cart" button
        • "Check Out" button
      • Data Entry: None
      • Buttons, Hyperlinks, Dropdowns:
        • Quantity dropdown
        • "Update Cart" button
        • "Check Out" button
      • Actions:
        • Clicking buttons
        • Navigating back to Shop Now page to update cart
      • Special Notes: I am not sure if the Shopify API (the payment processing and shopping API I plan to use for the website) requires me to build Checkout and payment info processing pages myself or if it provides them for the developer. Information will be shared as development progresses.
    • About Us:
      • Purpose: To provide the user with information about the founders of the business, as well as information on the business itself
      • Audience: Users who want background information on the business
      • Content:
        • Information on business founders
        • Information on the business's goals
        • Information on the business's purpose
      • Data Entry: None
      • Buttons, Hyperlinks, Dropdowns:
        • Hyperlink to Our Purpose page
        • Hyperlink to Our Goals page
        • Hyperlink to Our People page
      • Actions: Clicking the page links and navigating to each respective page
      • Special Notes: None
    • Specific Item Page:
      • Purpose: To give the user a more detail-rich view of the specific product clicked on
      • Audience: Users who want more information on a specific product
      • Content:
        • Picture gallery of specific product
        • Product description
        • "Add To Cart" button
        • Social media icons
        • Quantity dropdown
        • Review section
      • Data Validation: None
      • Buttons, Hyperlinks, Dropdowns:
        • Links to pictures in picture gallery
        • "Add To Cart" button
        • Social media icons
        • Quantity dropdown
      • Actions
        • Clicking buttons
        • Clicking icon links
        • Selecting quantity from dropdown
      • Special Notes: None
    • Our People (learn more):
      • Purpose: To give the user more information about the people that founded the business
      • Audience: Users who want more information on the business's founders
      • Content: Information on the founders of the business
      • Data Validation: None
      • Buttons, Hyperlinks, Dropdowns: None
      • Actions: None
      • Special Notes: None
    • Our Purpose (learn more):
      • Purpose: To give the user more information about the purpose of the business
      • Audience: Users who want more information on the business's purpose
      • Content: Information on the purpose of the business
      • Data Validation: None
      • Buttons, Hyperlinks, Dropdowns: None
      • Actions: None
      • Special Notes: None
    • Our Goals (learn more):
      • Purpose: To give the user more information about the goals of the business
      • Audience: Users who want more information on the business's goals
      • Content: Information on the goals of the business
      • Data Validation: None
      • Buttons, Hyperlinks, Dropdowns: None
      • Actions: None
      • Special Notes: None
  • Dynamic Functionality
    • Image carousel on Main Page (index), Ex. https://www.cms.k12.nc.us/Pages/default.aspx
    • Zooming in on images when hovering over them, Ex. Amazon Product Page
